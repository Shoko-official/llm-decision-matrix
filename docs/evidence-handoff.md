# Evidence Handoff

This note defines how future decision matrix criteria should reference research
ledger and taxonomy evidence.

It does not add decision claims, scores, rankings, recommendations, or
paper-ready wording.

## Handoff Requirements

Future decision matrix entries should identify:

* the related ledger claim ID, when applicable;
* the related ledger source ID or source IDs;
* the related taxonomy term or layer, when applicable;
* whether supporting evidence is primary or secondary;
* whether the criterion fit is ready for paper use;
* any unresolved evidence gap.

## Readiness States

| State | Meaning |
|---|---|
| `planning` | Entry is a placeholder or structure note. |
| `evidence_needed` | Entry needs approved ledger evidence before use. |
| `candidate` | Entry has evidence direction but is not paper-ready. |
| `ready` | Entry can support paper work after review. |
| `blocked` | Entry should not be used until the blocker is resolved. |

## Current Limits

Do not add final decision matrix scores, rankings, recommendations, or decision
rules from this handoff issue.
