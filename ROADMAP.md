# Roadmap

This roadmap starts with governance only. Decision matrix content starts after
Milestone 0 is complete and validation exists.

## Milestone 0: Project Governance

Goal: establish a small, reviewable operating model for the decision matrix
repository.

### Issues

1. [#1 Create decision matrix governance documentation](https://github.com/Shoko-official/llm-decision-matrix/issues/1)
2. [#2 Add issue and PR templates](https://github.com/Shoko-official/llm-decision-matrix/issues/2)
3. [#3 Add minimal validation, CI, and folder structure](https://github.com/Shoko-official/llm-decision-matrix/issues/3)

### Execution Order

1. Complete issue #1 before templates or validation.
2. Complete issue #2 before content changes.
3. Complete issue #3 before decision matrix content.

No matrix content should be added during Milestone 0.

## Acceptance Criteria

Milestone 0 is complete when:

* repository role is documented;
* roadmap exists;
* contribution and review rules exist;
* issue and PR templates exist;
* minimal validation commands exist;
* minimal CI exists;
* initial folders exist;
* no decision matrix content has been added.

## Later Milestones

Expected sequence:

1. Define a minimal matrix skeleton.
2. Define neutral decision criteria placeholders.
3. Add evidence handoff rules.
4. Add validation for matrix files.
5. Add tables only after evidence and criteria are ready.

Any change to matrix structure, criteria, evidence rules, or figures must happen
in a dedicated issue.
